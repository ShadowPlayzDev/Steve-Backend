Contact Us Module

A customizable and dynamically generated "Contact Us" section for your website, built with vanilla JavaScript, HTML, and styled with Tailwind CSS.

Features

  • Dynamic Content: All contact information (email, phone, address, social links) is dynamically added to the HTML by JavaScript.
  • Customizable Configuration: Easily update contact details, social media links, and module appearance via a simple JavaScript configuration file.
  • Strict Custom Configuration: If a custom configuration is provided, only the fields explicitly defined within it will be displayed. Missing fields will not fall back to defaults.
  • Default Fallback: If no custom configuration file is provided or loaded, the module gracefully falls back to a complete default set of contact information.
  • Themable: Supports light, dark, or app themes for the module's background and text colors, adapting to your website's design.
  • Random Social Icons: Social media icons are displayed in a random order each time the module loads.
  • Flexible Social Icon Display: Supports custom icon URLs (e.g., brand logos) or falls back to Font Awesome icons if a URL isn't provided.

Prerequisites

Before you begin, ensure you have:

  • A basic understanding of HTML.
  • Familiarity with JavaScript.
  • An internet connection to fetch the module's main script and Tailwind CSS CDN.

Setup Instructions

Follow these steps to integrate the Contact Us Module into your HTML page.

1. HTML Structure

Add the following HTML snippet to your webpage where you want the contact module to appear. This includes the div that will serve as the container and the two essential script tags.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website Contact</title>
    
    <!-- Link to Tailwind CSS CDN -->
    <script src="https://cdn.tailwindcss.com"></script>
    
    <!-- Configure Tailwind CSS for dark mode via 'class' strategy -->
    <script>
        tailwind.config = {
            darkMode: 'class', // Enables dark mode based on the 'dark' class on HTML or body
        }
    </script>
    
    <!-- Optional: Include Font Awesome for default social icons if you don't provide custom URLs -->
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0-beta3/css/all.min.css">
</head>
<body class="bg-gray-900 text-gray-100 min-h-screen flex items-center justify-center p-4">

    <!-- The Contact Us Module Container -->
    <!-- It has initial styling which will be overridden by the JS based on config.theme -->
    <div id="contact" class="p-6 rounded-lg shadow-xl max-w-md w-full transition-colors duration-300">
        <!-- Main module logic script -->
        <script type="module" src="https://steveow.vercel.app/custom-contact.js"></script>
        
        <!-- Your custom configuration script -->
        <!-- Replace './my-custom-config.js' with the actual path to your config file -->
        <script type="module" src="./my-custom-config.js"></script> 
    </div>

</body>
</html>

2. Tailwind CSS Setup

The module leverages Tailwind CSS for its styling. The HTML snippet above already includes the necessary CDN link for Tailwind and the configuration for darkMode: 'class'. This setup allows the module to dynamically change its theme based on your configuration.

  • The <body> tag uses bg-gray-900 and text-gray-100 to set a default dark theme for the entire page.
  • The <div id="contact"> container will have its specific background and text colors applied dynamically by the custom-contact.js script, according to the theme setting in your my-custom-config.js.

3. Create Your Custom Configuration File (my-custom-config.js)

Create a new JavaScript file (e.g., my-custom-config.js) in the same directory as your HTML file (or a subdirectory, adjusting the path in the HTML accordingly).

Paste the following content into my-custom-config.js and customize it with your contact details. If a field is omitted (e.g., you don't include "phone"), it simply won't be displayed by the module.

// my-custom-config.js
// This file defines the custom configuration for the Contact Us Module.

window.contactConfig = {
    // REQUIRED: Theme for the contact module.
    // Options: "light", "dark", "app"
    // - "light": Light background (gray-100), dark text (gray-900)
    // - "dark": Dark background (gray-800), light text (gray-100)
    // - "app": Attempts to blend with the surrounding page theme (defaults to dark for module background)
    theme: "light", 

    // OPTIONAL: Title for the contact section.
    // title: "Connect with Our Team", 

    // OPTIONAL: A short introductory message.
    // message: "We're always open to new connections. Find us on your preferred platform!", 

    // OPTIONAL: Contact email address.
    email: "info@yourcompany.com", 

    // OPTIONAL: Contact phone number.
    // phone: "+1 (123) 456-7890", 

    // OPTIONAL: Physical address.
    // address: "789 Tech Avenue, Suite 101, Cityville, CA", 

    // OPTIONAL: An array of social media links.
    // If this array is empty or not provided, no social icons will be displayed.
    socialIcons: [
        { 
            platform: "discord", 
            handle: "yourdiscordserverid", // Or your Discord username/tag
            icon: "https://cdn.discordapp.com/attachments/1247012674997092403/1247012693892749322/discord-logo-white-min.png" 
        },
        { 
            platform: "twitter", 
            handle: "MyCustomTwitter", 
            icon: "https://img.icons8.com/ios-filled/50/FFFFFF/twitter--v1.png" 
        },
        { 
            platform: "github", 
            handle: "MyCustomGitHub", 
            icon: "https://img.icons8.com/ios-filled/50/FFFFFF/github.png" 
        },
        { 
            platform: "linkedin", 
            handle: "your-linkedin-profile", // Usually the part after /in/
            // No 'icon' provided here, will use Font Awesome if linked in HTML
        },
        { 
            platform: "instagram", 
            handle: "yourinstagramhandle", 
            icon: "https://img.icons8.com/ios-filled/50/FFFFFF/instagram-new.png" 
        }
        // Add more social links as needed
    ]
};

4. Understand custom-contact.js

The main module logic is located at https://steveow.vercel.app/custom-contact.js. You generally do not need to modify this file directly. It's responsible for:

  • Loading your custom configuration (if provided). If no custom config is detected, it falls back to a complete default configuration.
  • Applying the correct theme classes to the module's container based on your configuration.
  • Dynamically generating only the contact elements (email, phone, address, etc.) that are explicitly defined in your custom configuration.
  • Randomizing the order of social icons.
  • Logging messages to the console for configuration loading status.

Configuration Options

Here's a detailed breakdown of the properties you can set in your my-custom-config.js file:

  • theme (String, REQUIRED):
    • "light": The module will have a light background (bg-gray-100) and dark text (text-gray-900).
    • "dark": The module will have a dark background (bg-gray-800) and light text (text-gray-100).
    • "app": The module attempts to blend with the general app theme. In the current implementation, this defaults to the dark module styling if no global theme is applied.
  • title (String, OPTIONAL): The main heading for your contact section (e.g., "Get in Touch"). If omitted, no title will be displayed.
  • message (String, OPTIONAL): A short descriptive text or welcome message. If omitted, no message will be displayed.
  • email (String, OPTIONAL): An email address. It will be rendered as a clickable mailto: link. If omitted, no email link will be displayed.
  • phone (String, OPTIONAL): A phone number. It will be rendered as a clickable tel: link. If omitted, no phone link will be displayed.
  • address (String, OPTIONAL): A physical address. If omitted, no address will be displayed.
  • socialIcons (Array of Objects, OPTIONAL):
    • This is an array where each object represents a social media link. If this array is empty or not provided, no social icons will be displayed.
    • Each object must have:
      • platform (String): The name of the platform (e.g., "discord", "twitter", "linkedin", "github", "instagram", "facebook"). This is used for generating the link and as fallback text if no icon is provided.
      • handle (String): The username, ID, or specific path component for that platform (e.g., shadowplayzdev for Discord, your-profile-name for LinkedIn).
    • Each object can also have:
      • icon (String, OPTIONAL): A full URL to an image file (e.g., SVG, PNG) that will be used as the icon for that social link. If not provided, the module will attempt to use a Font Awesome icon based on the platform name (requires Font Awesome CDN to be linked in your HTML).

Social Icons Compatibility

The module provides predefined URL structures for common social platforms (Twitter, LinkedIn, GitHub, Discord, Instagram, Facebook). For other platforms:

  • If you provide a platform name not explicitly listed, the module will try to construct the link by prepending https:// to your handle, or use the handle directly if it already starts with http.
  • Recommendation: For less common social sites, it is best to provide the full URL to your profile or page directly within the handle property. For example: { platform: "myblog", handle: "https://www.myblog.com/myprofile" }.
  • Icon display via img src with a custom icon URL should work for any image type supported by browsers.
  • Fallback to Font Awesome icons requires the Font Awesome CDN to be included in your HTML. If neither a custom icon nor Font Awesome is available, only the platform name will be displayed as text.

How to Use

  1. Save the HTML: Save the HTML code from Section 1 as an .html file (e.g., index.html).
  2. Create Config File: Create your my-custom-config.js file as described in Section 3, customizing it with your information. Ensure it's in the correct relative path from your HTML file.
  3. Deploy: Upload your index.html and my-custom-config.js files to your web server (e.g., GitHub Pages, Vercel, Netlify, or any traditional hosting).
  4. View: Open your webpage in a browser. The contact module should appear, dynamically populated only with your configured details. Check your browser's developer console for any messages about configuration loading.
Important: If you omit an optional field (like phone or address) in your my-custom-config.js, it will simply not be displayed by the module. The default configuration will only be used if window.contactConfig is not detected at all (e.g., if you remove the second script tag entirely or it points to a non-existent file).
